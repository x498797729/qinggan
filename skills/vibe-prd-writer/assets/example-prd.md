# 智能笔记助手 PRD

## 产品定位

一个 AI 驱动的笔记工具，帮你在30秒内把会议文本整理成结构化的摘要、待办事项和决策记录。

## 目标用户

经常开会的产品经理和项目管理者，需要在会议后快速整理记录，但总是花费大量时间且容易遗漏关键信息。他们通常在办公室使用电脑，对技术要求不高，只要简单易用就行。

## 核心功能（最多3个）

### Feature 1: 自动提取关键信息

**用户操作：** 粘贴或上传会议文本/录音转文字
**预期结果：** 系统自动识别并提取：会议摘要、待办事项、决策记录
**验收标准：** 5 页以内的会议文本，在 5 秒内完成提取，待办事项准确率 > 90%

### Feature 2: 待办事项智能分配

**用户操作：** 确认或修改 AI 提取的待办事项
**预期结果：** 每个待办事项自动标记：负责人、截止时间、优先级
**验收标准：** 用户能在 30 秒内完成一次待办事项的确认和分配

### Feature 3: 决策时间线

**用户操作：** 查看按时间顺序排列的所有决策
**预期结果：** 能快速回溯"谁在什么时候说了什么决定"
**验收标准：** 决策按时间倒序排列，点击任意决策能看到上下文

## 明确不做（v1）

- 不做实时语音转录（需要用户上传文本或录音文件）
- 不做多语言支持（v1 只做中文）
- 不做团队协作和分享功能
- 不做数据分析和洞察建议
- 不做移动端 App

## 技术方案

| 组件 | 技术选型 | 理由 |
|------|---------|------|
| 前端 | Next.js + Tailwind CSS | Next.js 前后端一体，Tailwind 写 UI 超快，两者组合是 vibe coding 的黄金搭档 |
| AI 能力 | Claude API (claude-3-5-sonnet) | 文本理解能力强，输出稳定，性价比高，适合内容提取场景 |
| 数据存储 | LocalStorage | 单用户场景，无需后端存储，降低复杂度，本地隐私更好 |
| 部署 | Vercel | 免费额度够用，一键部署，和 Next.js 天生一对 |

## 产品风格

- **UI 风格：** 工具化、清晰直接、像你想象中的高效工具
- **语言：** 中文为主
- **色调：** 浅色为主，强调信息的可读性

## 给 Agent 的执行指令

> **执行节奏：**
> 1. 先搭 Next.js 基础框架和页面结构（首页、输入页、结果页）
> 2. 实现核心功能：AI 文本提取（Feature 1）
> 3. 实现待办事项界面和确认功能（Feature 2）
> 4. 实现决策时间线页面（Feature 3）
> 5. 用 Tailwind 做基础样式，确保响应式
> 6. 自测：粘贴几段会议文本，验证提取结果是否符合验收标准
>
> **关键决策：**
> - 数据存在 LocalStorage，不需要后端数据库
> - AI 调用要在 API Routes 里做，不要暴露 API Key
> - 错误处理要友好，AI 提取失败时让用户能手动编辑
>
> **遇到问题时：**
> - 如果某个功能实现复杂，优先选最简单的方案
> - 不确定 UI 怎么做，就照着 Notion 的感觉来，简洁好用
