# Vibe PRD Writer

一个像 AI PM 朋友一样的 Skill，帮你把模糊的想法变成 AI 能直接执行的一页纸 PRD。

## 🎯 适用场景

你用 Claude Code / Cursor 做 vibe coding 时，需要把想法说清楚给 AI 执行。

**触发方式：**
- "帮我写个PRD"
- "我想做一个XX"
- "帮我梳理需求"
- "我要vibe code一个产品"

## 🤖 Skill 怎么帮你

### Step 1: 判断你的状态

先看看你的想法是模糊还是清晰，告诉你接下来怎么走。

- **想法模糊** → 一步步引导你思考
- **想法清晰** → 快速确认几个关键点直接出 PRD

### Step 2: 5 个必问问题

每次只问一个，等回答了再问下一个。每个问题都会告诉你"为什么要问这个"。

1. **为什么要做这个？** — 动机决定了后面的一切
2. **给谁用？** — 具体用户场景决定所有决策
3. **核心功能是什么？** — 砍需求是产品经理的核心技能
4. **明确不做什么？** — 没有边界的 PRD 会失控
5. **怎么算成功？** — 没有验收标准的 PRD 就是许愿清单

### Step 3: 技术方案建议

根据产品类型自动推荐合适的技术栈：

| 产品类型 | 技术方案 | 理由 |
|---------|---------|------|
| 纯展示类 | HTML + Tailwind + Vercel | 零后端，秒部署 |
| AI 交互 | Next.js + AI API + Vercel | Next.js 处理 API，Vercel 免费够用 |
| 需要存数据 | Next.js + Supabase + Vercel | Supabase 一站式数据方案 |
| 不确定 | Next.js + Tailwind + Vercel | 默认黄金组合，兼容大部分情况 |

### Step 4: 生成一页纸 PRD

输出包含：
- 产品定位（一句话）
- 目标用户（2-3 句话）
- 核心功能（最多3个，含验收标准）
- 明确不做（本版本排除什么）
- 技术方案（附选型理由）
- 产品风格（UI、语言、色调）
- 给 Agent 的执行指令（最关键）

### Step 5: 质量自检

自动检查：
- ✓ 能否一句话说清楚产品是什么？
- ✓ MVP 范围一个下午能做完吗？
- ✓ "明确不做"列了吗？
- ✓ 每个功能有验收标准吗？
- ✓ 技术方案匹配需求吗？
- ✓ 这份 PRD 直接给 AI，它能立刻开始工作吗？

## 📦 安装

```bash
cp -r vibe-prd-writer ~/.claude/skills/
```

然后重启 Claude Code。
